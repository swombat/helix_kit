It would be nice to have a way to search through conversations to find a specific message and conversation.

Let's keep it simple for now - just straight search through all the messages in a given account, for the specific word (case insensitive).

The search box should be in the nav bar.

When searching, the main content area should then display the results, if any. Each result should include the line before and line after the search term, from the matching message.

If the user clicks on the message, they are taken to the conversation in question. Since there is no easy way to go to the message itself, just go to the conversation.

## Clarifications

- **Search trigger**: Submit-based (press Enter or click search icon), not live as-you-type.
- **Search scope**: Message content only (not chat titles or summaries).
- **Results display**: Dedicated search results page at a new route (e.g., `/accounts/:id/chats/search?q=term`), not a dropdown or overlay.
- **Data model note**: The app uses "Chats" internally (not "Conversations"). Messages belong to Chats, which belong to Accounts.
- **Case insensitive**: As stated in requirements, search is case insensitive.
- **Context**: Each result shows the line before and after the matching term from the message content.
- **Navigation**: Clicking a result navigates to the chat (not the specific message).
