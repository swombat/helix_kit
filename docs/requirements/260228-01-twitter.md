Nexus has a twitter account.

How can we give the agents a tool to post to Twitter?

And maybe a tool to receive messages from Twitter and respond to them?

Investigate the options for doing this, then ask clarifying questions, then present some possible options for implementation. Then plan them.

## Clarifications

1. **Account model**: One shared X/Twitter account for all Nexus agents. All agents post from the same account.
2. **Scope**: Post-only for v1 — no inbound mentions or DM reading. Just the ability to tweet.
3. **Moderation**: Auto-post with audit log. Agents post directly but all tweets are logged and reviewable. No human-in-the-loop approval queue.
4. **Credentials**: Per-account database config (encrypted) so different Nexus accounts could have different X integrations in the future.

## Research Findings

- **Recommended gem**: `x` (sferik/x-ruby) — actively maintained, supports X API v2
- **API tier**: Free tier allows 1,500 tweets/month, sufficient for v1
- **Auth**: OAuth 1.0a required for posting (user-context operations)
- **Rate limits**: ~300 posts per 3-hour window, 1,500/month on free tier
- **Credentials needed**: API Key, API Secret, Access Token, Access Token Secret
